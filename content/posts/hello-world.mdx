Welcome to my blog! This first post doubles as documentation for how the site itself works.

## The stack

Everything here is statically generated from `.mdx` files living in the repository:

- **Next.js 16** (App Router) for routing and static generation
- **TypeScript 6** in strict mode
- **Tailwind CSS 4** for styling
- **MDX** for content — Markdown with the option to embed React components

## Why files instead of a CMS?

As a backend engineer, I want my content pipeline to behave like code:

```typescript
export function getPostBySlug(slug: string): Post {
  const raw = fs.readFileSync(`content/posts/${slug}.mdx`, "utf8");
  const { data, content } = matter(raw);
  return { slug, ...data, content };
}
```

Posts are version-controlled, reviewable, and greppable. Publishing is `git push`. There is no database to migrate, no admin panel to secure, and nothing to pay for.
